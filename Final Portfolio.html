Final Portfolio

Introduction

AMS autonomous mobility scooter is designed primarily for elderly, also the elderly diseases are also considered in designing their applications such as big icons and texts for presbyopia, medication reminders for dementia. Some special features to help users monitoring their health consideration (heartbeat, blood pressure etc.).

Design concept



Through preliminary user surveys (interviews and questionnaires), we draw out the usage of our products and the usage scenarios, that is, the applicable population is elderly, and the use scenarios are streets, airports, and hospitals. Here, we find that users mostly like places that don't cause crowding and are not too dangerous.

Figure 1.1 - design concept

Design Process

After the background investigation, we thought of its expansion, the scooter software - the software installed in the scooter tablet. A software that better protects the applicable population (elderly), including control speed, maps, health management, clinics, etc. When using a mobility scooter, we can't automate the first step with just the button. We hope that the scooter is more convenient and more practical, because our main audience is the elderly people, so we designed a software for the elderly to give it some special features, such as helping users measure heartbeat blood pressure and statistical walking steps, and can send it to doctors and family members at any time. Sometimes older people may not be able to see the things clearly, so a voice service is established. In many cases, elderly are not good at using mobile phones, so the family often does not know the location of the elderly. We can use the scooter software to turn on the positioning settings so that the family can know the location of the old man at any time.


Sketch

We designed a software for the more convenient use of the scooter. According to the previous user survey, we made two softwares for the airport scene and the street scene.


According to the online research report, we have enlarged all the icons and fonts (compared to the normal software) to make our software more suitable for the elderly. Since one is for the street one is for the airport, the software has different functions. In the street, we mainly promote health care, for example, we can measure heartbeat blood pressure at any time. We can go directly to the default clinic through software and save doctors and drug records in the clinic. At the airport, our main push is to store luggage at any time, etc.

In fact, we have some ambiguity about the font change. The change of the font makes the overall interface not beautiful enough, but in the end we still use this function, because this software is designed for the elderly.

We selected some colors suitable for the elderly to watch according to the online research report.

Map-If the map can be enlarged, it can make the user see more clearly, and add the positioning function to let the user know the current location.

Because of the 2D colour used by most of the interface, the gradient colour of the "help" key seems too abrupt, and the user suggests using a slightly lighter colour. Because the icon is too small, the colour of the icon is too bright and there is no primary colour style that makes it difficult for users with presbyopia to use. The button lines of the page turning are very thin, and the user feels that it is not obvious enough. Separating the date and time makes the user feel uncomfortable

We designed the “help” button with a gradient colour to make it obvious for user, so we changed it with a darker colour, but it is still a gradient colour. The icons changed to bigger vision with soft pastel colours and the date is put next to the time.

Refelection

We change our model based on existing online reports and user interviews.But I think our software features are great for older people, and we search a lot of reports and user surveys to find out which ones are best for older people.

Design style

Using too many different colors. Our original intention is to let the elderly people get visual reminders through the color, but too many colors have caused visual clutter, we should probably think about it how to solve this problem.

Interactive experience

Due to limited time, we didn't open all the features in the software, and I think we might be able to connect our software to some existing websites like google and facebook.
